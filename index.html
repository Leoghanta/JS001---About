H17334 - Developing Software

About Developing Software

This is an introductory programming module that will enable you to develop the basic knowledge of software development skills. We will be looking at JavaScript which is an introductory language to software development, but also pertinent to the web development modules you will be working on.

H17334

This is your SQA unit code. This module will credit you with 1 HN credit at SCQF Level 7. This is a core module which means you need this module to pass your HNC Digital Design and progress to the HND.

This module has two learning outcomes:

Outcome 1

Implement and test code to carry out tasks following a given design.

  1. Make appropriate use of variables
  2. Use programming constructs (Selection and Iteration) within code
  3. Use functions and parameter passing
  4. Test the code using a recognised test strategy and given test cases
  5. Document the result of testing

Outcome 2

Prepare technical documentation in line with good practice

  1. Apply meaningful names to variables and functions
  2. Ensure that code is well laid out to aid maintainability
  3. Internally document the code by adding comments
  4. Produce technical guide

More details can be found from the SQA

Why JavaScript?

  1. ✅ It is one of the world's most popular programming languages.
  2. ✅ Javascript is the programming language of the Web.
  3. ✅ JavaScript is easy to learn.
  4. ✅ You can use this module in conjunction with your other modules to build interactive web sites.

JavaScript can also springboard you into further learning for Node.js, React.js, AJAX, Vue, Angular, Nuxt, Ember, etc. The JavaScript we will be learning in class will be client-side. HND will cover more programming for server-side programming. When you are confident with front-end development (client-side) and back-end development (server-side) we call this full stack development.

Assessments

This module has two outcomes which will be assessment into one assessment. The assessment will contain a web-based problem where you will need to follow a design and build the JavaScript code to make it work. The code will then go through a thorough testing phase and you will produce technical documentation to acompany your finished product.

What will this semester look like?

The roadmap of your semester is below.

In weeks 1 to 3, we will look at the JavaScript code and you will be able to build code snippets and getting to know the structure of the language.

In weeks 4 to 6, we will look at structuring our code, looking at code flow, and organising the code into programs. By week 6, you will be a JavaScript programmer.

In weeks 7 to 9, we will bring our programming style to the Document Object Model to create interactive web pages.

In weeks 10 to 12, we will look at the Software Development Life Cycle (SDLC) with a view of building software and fully testing it for deployment.

Lastly, the last four weeks will be reserved for the assessment.
